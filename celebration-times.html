Celebration times

This page documents what triggers a "celebration" in the Fun Times clocks app.

Currently implemented triggers

  • Manual test — The "Test Celebration" button in the debug panel (on funtimes.html) triggers a manual celebration for testing.
  • No automatic scheduled celebrations yet — The current funtimes.html does not include any built‑in scheduled celebration rules.

How to add scheduled celebrations (recommended)

Use a JSON file to list times you want to celebrate. Place it at data/celebrations.json and load it from the clocks page.

Example format (simple list of exact times in 24‑hour HH:MM or HH:MM:SS):

[
  "00:00",        // midnight
  "12:00",        // noon
  "07:07:07",     // a specific second-precise time
  "13:37"         // 13:37 every day
]

Suggested integration (JS)

On each tick compare the current HH:MM or HH:MM:SS to entries from the JSON and fire your celebration routine when they match. Example (pseudo):

// load JSON once, then on each tick:
const now = new Date();
const hhmm = now.toTimeString().slice(0,5);      // "HH:MM"
const hhmmss = now.toTimeString().slice(0,8);    // "HH:MM:SS"
if (celebrationList.includes(hhmm) || celebrationList.includes(hhmmss)) {
  triggerCelebration();
}

Notes

  • Decide whether matches should be second-precise or minute-precise and format your JSON accordingly.
  • Consider time zone and system clock drift if you need precise synchronization.
  • For recurring patterns (every hour, every 15 minutes) implement pattern logic in JS rather than storing every occurrence in JSON.
